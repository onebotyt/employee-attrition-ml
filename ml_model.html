Step 1
Preprocessing — Getting Data Ready
Before any ML model can learn, all data must be converted to numbers and put on the same scale. These helper classes do that work.
🔤 LabelEncoder — Converts Text to Numbers

ML models only understand numbers. LabelEncoder converts text columns like "Yes"/"No" or "Sales"/"R&D" into 0, 1, 2...

ml_models.py — LabelEncoder
class LabelEncoder:
    def __init__(self):
        self.classes_ = None   # will store all unique text values found
        self._map = {}           # text → number  e.g. {"No":0, "Yes":1}
        self._inv = {}           # number → text  e.g. {0:"No", 1:"Yes"}

    def fit(self, y):
        self.classes_ = np.array(sorted(set(y)))         # find all unique values, sort them
        self._map = {v: i for i, v in enumerate(self.classes_)} # assign number to each
        self._inv = {i: v for v, i in self._map.items()}  # reverse mapping
        return self

    def transform(self, y):
        return np.array([self._map[v] for v in y])  # replace each text with its number
1
set(y) — Find all unique values
If column has ["No","Yes","Yes","No"] → set gives {"No","Yes"}
2
sorted(...) — Sort alphabetically
Sorted ["No","Yes"] → "No" gets 0, "Yes" gets 1 (always consistent)
3
enumerate(classes_) — Assign numbers
Creates _map = {"No":0, "Yes":1} and _inv = {0:"No", 1:"Yes"}
💡 Real example: OverTime column ["Yes","No","Yes","Yes"] becomes [1, 0, 1, 1] — the model can now do math on it.
⚖️ StandardScaler — Put All Numbers on the Same Scale

Age ranges 18–60. Monthly Income ranges 1000–20000. If we don't scale, Income dominates because its numbers are much bigger. StandardScaler makes every feature have mean = 0 and std = 1.

The Formula
scaled_value = (original_value − mean) / std
Example: Age mean=35, std=10. Age of 45 → (45−35)/10 = 1.0
Age of 25 → (25−35)/10 = −1.0
After scaling: 0 means average, +1 means 1 standard deviation above average
ml_models.py — StandardScaler
class StandardScaler:
    def fit(self, X):
        self.mean_ = X.mean(axis=0)       # average of each column across all rows
        self.std_  = X.std(axis=0)        # spread of each column
        self.std_[self.std_ == 0] = 1.0  # if all values are same, std=0 → avoid divide by zero
        return self

    def transform(self, X):
        return (X - self.mean_) / self.std_  # apply formula to every value
🔴 Critical rule: Always call fit_transform(X_train) on training data, then only transform(X_test) on test data. Never fit on test data — that would be data leakage!

Step 2
Evaluation Metrics — How We Measure Model Quality
After the model makes predictions, we need numbers to say "how good is it?" These functions compute those numbers.

CONFUSION MATRIX — The Foundation of All Metrics

FN = employee who will leave but we predicted they'd stay — the most costly mistake!

📏 Accuracy
Accuracy = (TP + TN) / Total
How many predictions were correct overall
Problem: If 83% stay, always saying "stay" = 83% accuracy but useless! 
def accuracy_score(y_true, y_pred):
    return np.mean(y_true == y_pred)
# == creates array of True/False, mean gives fraction of True
⭐ Recall (PRIMARY METRIC)
Recall = TP / (TP + FN)
Of all employees who actually left, how many did we catch?
High Recall = few missed leavers — most important for attrition! 
def recall_score(y_true, y_pred, pos=1):
    tp, _, fn = _tp_fp_fn(y_true, y_pred, pos)
    return tp / (tp + fn) if (tp+fn) > 0 else 0
🎯 Precision
Precision = TP / (TP + FP)
Of everyone we flagged as "will leave", how many actually left?
Low precision = many false alarms (unnecessary retention offers)
⚡ F1 Score
F1 = 2 × (Precision × Recall) / (P + R)
Harmonic mean of Precision and Recall.
One number that balances both — useful for imbalanced data.
📉 ROC-AUC — Model Quality Across All Thresholds

AUC (Area Under Curve) measures how well the model separates "will leave" from "will stay" at every possible threshold. AUC = 1.0 = perfect. AUC = 0.5 = random guessing.

def roc_auc_score(y_true, y_score):
    fpr, tpr, _ = roc_curve(y_true, y_score)

📈
Logistic Regression
Algorithm 1 of 5 · Best Recall in our project · Simplest ML classifier
💡 What it does in simple words: Imagine drawing a line through your data that separates "will leave" employees from "will stay" employees. Logistic Regression finds the best possible line (or plane in higher dimensions). It then outputs a probability between 0 and 1 for each employee.
🌊 Step 1 — The Sigmoid Function (Core of Logistic Regression)
Any number → squashed to a probability between 0 and 1. This is the secret weapon of logistic regression.
Sigmoid Formula
sigmoid(z) = 1 / (1 + e^(-z))
z = 0 → sigmoid = 0.5 (50% chance of leaving)
z = 5 → sigmoid ≈ 0.99 (99% chance of leaving)
z = -5 → sigmoid ≈ 0.01 (1% chance of leaving)
ml_models.py — _sigmoid()
def _sigmoid(self, z):
    return 1 / (1 + np.exp(-np.clip(z, -500, 500)))
    #                        ↑ clip prevents overflow: e^(-1000) = inf in Python
    #                          clip keeps z between -500 and 500 before exp
🔢 Step 2 — The Linear Equation (z = X · w + b)
Every feature (Age, Income, OverTime...) gets multiplied by a weight. The model learns these weights during training.
The Decision Equation
z = (Age × w₁) + (Income × w₂) + (OverTime × w₃) + ... + b
In code: z = X @ self.coef_ + self.intercept_
@ is matrix multiplication — computes all features × weights at once for all rows
self.coef_ = the weights (one per feature, 25 total)
self.intercept_ = bias term b (shifts the decision boundary)
⛰️ Step 3 — Gradient Descent (How the Model Learns Weights)
The model starts with random weights, makes predictions, measures error, then adjusts weights to reduce error. Repeated 500 times.

Each epoch: calculate gradient (slope), take a small step downhill → weights improve

ml_models.py — fit() Gradient Descent Loop
def fit(self, X, y):
    n, d = X.shape                           # n = num employees, d = num features (25)
    self.coef_ = rng.normal(0, 0.01, d)      # start with tiny random weights near 0
    self.intercept_ = 0.0                    # bias starts at 0
    sw  = self._sample_weights(y)            # higher weight for minority class (leavers)
    lam = 1.0 / (self.C * n)                # regularization strength (prevents overfitting)

    for epoch in range(self.max_iter):        # repeat 500 times
        lr_t = self.lr / (1.0 + 0.01 * epoch)  # FIXED: learning rate shrinks over time
        z    = X @ self.coef_ + self.intercept_  # predict scores for all employees
        p    = self._sigmoid(z)               # convert scores to probabilities (0-1)
        err  = (p - y) * sw                    # error = predicted - actual, weighted by class
        grad = X.T @ err / n + lam * self.coef_  # gradient = how much to adjust each weight
        self.coef_      -= lr_t * grad         # update weights (move downhill)
        self.intercept_ -= lr_t * err.mean()  # update bias
1
rng.normal(0, 0.01, d) — Random starting weights
Start near 0 so no feature has unfair advantage. If we start too large, gradient can explode.
2
lam = 1.0 / (C * n) — Regularization
Prevents overfitting. Higher C = less regularization (trust the training data more). Lower C = more regularization (simpler model).
3
lr_t = self.lr / (1.0 + 0.01 * epoch) — Adaptive learning rate (FIX)
Learning rate shrinks each epoch. Early: big steps to find the right direction. Later: tiny steps to fine-tune. Prevents overshooting the minimum.
4
err = (p - y) * sw — Weighted error
sw (sample weight) is larger for leavers (minority class). This forces the model to pay more attention to getting leavers right.
5
grad = X.T @ err / n + lam * self.coef_ — Gradient
X.T @ err = how each feature contributed to the error. Divided by n to average. Plus regularization term.

🌳
Decision Tree
Algorithm 2 of 5 · Used internally by Random Forest · Most explainable algorithm
💡 What it does in simple words: Think of 20 questions game. "Does the employee work overtime?" → Yes → "Is monthly income below $3000?" → Yes → "Will Leave" with 78% probability. The tree learns which questions to ask and in what order.
📐 Gini Impurity — Measuring "Mixedness" of a Group
Before splitting, we need to measure how mixed a group is. A pure group (all Stay or all Leave) is better for predictions than a mixed group.
Gini Impurity Formula
Gini = 1 − p₁² − p₀²
p₁ = fraction of employees who Left, p₀ = fraction who Stayed
Pure group (100% Leave): Gini = 1 − 1² − 0² = 0.0 (perfect, no mixing)
Mixed group (50% each): Gini = 1 − 0.5² − 0.5² = 0.5 (worst case)
ml_models.py — _gini()
def _gini(self, y, w):
    total = w.sum()               # total weight in this node
    if total == 0: return 0
    p1 = (w[y == 1]).sum() / total  # weighted fraction of "Will Leave"
    return 1 - p1**2 - (1-p1)**2   # gini formula
    # w = sample weights (leavers count more due to class_weight='balanced')
✂️ Finding the Best Split — _best_split()
For every feature and every possible threshold, the tree tries the split and measures how much it reduces Gini impurity. The split with the biggest reduction wins.
ml_models.py — _best_split()
def _best_split(self, X, y, w):
    g_parent = self._gini(y, w)               # how impure is the current node?
    best_gain = -1

    for f in feats:                            # try each feature (random subset for RF)
        vals = np.unique(X[:, f])              # all unique values of this feature
        thresholds = (vals[:-1] + vals[1:]) / 2  # try midpoints between each pair

        for thr in thresholds:                 # e.g. try Income < 5000, 5500, 6000...
            left  = X[:, f] <= thr             # employees below threshold
            right = ~left                      # employees above threshold

            g_l = self._gini(y[left],  w[left])   # impurity of left group
            g_r = self._gini(y[right], w[right])  # impurity of right group
            wl, wr = w[left].sum(), w[right].sum()
            gain = g_parent - (wl*g_l + wr*g_r) / (wl+wr)  # weighted avg impurity decrease

            if gain > best_gain:               # keep the best split seen so far
                best_gain, best_feat, best_thr = gain, f, thr

    return best_feat, best_thr               # return the winning split
🌿 Building the Tree — _build() (Recursive)
The tree builds itself by splitting, then splitting again on each side. It stops when data is pure, too small, or maximum depth is reached.

Each split divides employees into 2 groups. Leaf nodes return the probability of leaving.

ml_models.py — _build() recursive tree construction
def _build(self, X, y, w, depth):
    node = _Node()
    node.weight = w.sum()                           # total weight of all employees here
    node.gini   = self._gini(y, w)                  # store gini for feature importance

    # STOP CONDITIONS — make this a leaf node
    if (depth >= self.max_depth                      # too deep → stop (prevents overfitting)
            or len(y) < self.min_samples_split       # too few samples → stop
            or len(np.unique(y)) == 1):             # all same class → already pure → stop
        node.value = self._leaf_value(y, w)           # store probability of leaving
        return node

    feat, thr = self._best_split(X, y, w)           # find best question to ask
    if feat is None:                                 # no useful split found
        node.value = self._leaf_value(y, w)
        return node

    mask = X[:, feat] <= thr                          # split employees into left/right
    node.left  = self._build(X[mask],  y[mask],  w[mask],  depth+1)  # recurse left
    node.right = self._build(X[~mask], y[~mask], w[~mask], depth+1) # recurse right
    return node

Random Forest
Algorithm 3 of 5 · 200 Decision Trees voting together · Best feature importance
💡 What it does in simple words: Instead of one decision tree, we build 200 trees. Each tree is trained on a slightly different random sample of the data and uses random subsets of features. Final prediction = majority vote of all 200 trees. Many imperfect trees together beat one perfect tree.
🎲 Step 1 — Bootstrap Sampling (Bagging)
Each tree gets a different random sample of the training data (with replacement — same employee can appear multiple times). This ensures variety between trees.
ml_models.py — RandomForest.fit() Bootstrap Loop 
def fit(self, X, y):
    n, d = X.shape
    max_feat = max(1, int(np.sqrt(d)))          # each tree sees √25 ≈ 5 random features
    sw = self._sample_weights(y)                  # class weights for imbalanced data

    for i in range(self.n_estimators):             # build 200 trees
        idx = rng.choice(n, size=n, replace=True)  # random sample WITH replacement
        #   ↑ e.g. from 800 employees, pick 800 (some appear 2-3x, some 0x)
        Xi, yi, wi = X[idx], y[idx], sw[idx]         # get that tree's training data

        tree = _DecisionTree(
            max_features=max_feat,                    # only √d features per split
            random_state=int(rng.integers(0, 99999)) # different seed → different splits
        )
        tree.fit(Xi, yi, wi)                        # train this tree on its sample
        self.trees_.append(tree)                    # add to the forest
🗳️ Step 2 — Ensemble Voting (All Trees Vote)
For each employee, all 200 trees output a probability. We average all probabilities. This smooths out individual tree mistakes.
ml_models.py — predict_proba() Voting
def predict_proba(self, X):
    probas = np.mean([t.predict_proba(X) for t in self.trees_], axis=0)
    # List comprehension: get each tree's probability for all employees
    # shape: (200_trees, 200_employees, 2_classes)
    # np.mean(..., axis=0): average across all 200 trees
    # Result shape: (200_employees, 2_classes) — one averaged probability per employee
    return probas
🔑 Step 3 — Feature Importance (FIXED with Correct Gini Formula)
After training, we calculate which features were most useful for splitting across all 200 trees. The old code used sample count (wrong). The fixed code uses weighted Gini reduction (correct — same as sklearn).
Correct Feature Importance Formula
importance = Σ (n_node/N) × (gini_parent − wL×giniL/n − wR×giniR/n)
For each split node: how much did this split reduce impurity, weighted by how many employees passed through it?
Sum across all trees, then normalise to sum to 1.
ml_models.py — _walk() correct Gini importance (FIXED)
def _walk(self, node, scores, n_total):
    if node is None or node.value is not None: return  # skip leaf nodes

    n_node  = node.weight                        # how many employees in this node
    n_left  = node.left.weight                   # employees going left
    n_right = node.right.weight                  # employees going right

    impurity_decrease = (n_node / n_total) * (
        node.gini                                # impurity before split
        - (n_left  / n_node) * node.left_gini    # minus weighted left impurity
        - (n_right / n_node) * node.right_gini   # minus weighted right impurity
    )                                            # = net reduction in impurity from this split

    scores[node.feat] += max(0.0, impurity_decrease)  # add to this feature's score
    self._walk(node.left,  scores, n_total)      # recurse into left subtree
    self._walk(node.right, scores, n_total)      # recurse into right subtree

# OLD (WRONG):  scores[node.feat] += node.weight  ← just sample count, not impurity!

Result: MonthlyIncome, Age, YearsAtCompany now correctly show as top drivers because they reduce impurity the most — not just because many employees pass through those nodes.

K-Nearest Neighbours (KNN)
Algorithm 4 of 5 · No training phase · Distance-based · Simplest concept
💡 What it does in simple words: To predict if Employee X will leave, find the 5 most similar employees in the training set (nearest neighbours). If 4 out of 5 of them left, predict "Will Leave" with 80% probability. "Tell me who your neighbours are and I'll tell you what you'll do."
📏 Step 1 — Euclidean Distance
We measure similarity between two employees using Euclidean distance — same as Pythagorean theorem but in 25 dimensions.
Distance Formula (Euclidean)
distance = √( (a₁−b₁)² + (a₂−b₂)² + ... + (a₂₅−b₂₅)² )
Compare two employees across all 25 features. Small distance = very similar employees.
ml_models.py — _distances()
def _distances(self, x):
    diff = self.X_train_ - x              # subtract query employee from ALL training employees
    #  Result: matrix where each row is (train_employee − query_employee) for all 25 features
    return np.sqrt((diff**2).sum(axis=1))  # square each diff, sum across features, sqrt
    # No explicit for loop — NumPy does this for all 800 training employees at once (fast!)
⚖️ Step 2 — Distance-Weighted Voting (FIXED)
Old code: every neighbour gets 1 vote (uniform). Fixed code: closer neighbours get more weight. If nearest neighbour is very similar, their vote counts more.
ml_models.py — predict_proba() Fixed KNN voting 
def predict_proba(self, X):
    probas = []
    for x in X:                               # for each new employee
        dists = self._distances(x)           # distance to all 800 training employees
        nn    = np.argsort(dists)[:k]        # indices of 5 closest employees
        nn_y  = self.y_train_[nn]            # did those 5 employees leave? (0 or 1)
        nn_d  = dists[nn]                     # their distances

        # OLD (WRONG): p1 = nn_y.mean()  ← simple majority, ignores distance

        # FIXED: inverse distance weighting
        w = 1.0 / (nn_d + 1e-8)             # closer → larger weight (1e-8 prevents ÷0)
        # e.g. distance 0.1 → weight 10.0 (very similar)
        #      distance 2.0 → weight 0.5  (not very similar)

        cw = [self._cw.get(yi) for yi in nn_y]  # class weights (leavers count more)
        w  = w * cw                           # combine distance + class weights

        p1 = np.sum(w[nn_y == 1]) / w.sum()  # probability = weighted fraction who left
        probas.append([1 - p1, p1])
    return np.array(probas)

SVM — Support Vector Machine (Linear)
Algorithm 5 of 5 · Finds maximum margin boundary · Uses SGD training
💡 What it does in simple words: Logistic Regression finds "a line that separates the two classes". SVM finds "the widest possible street between the two classes". The goal is to maximize the margin (gap) between the nearest "Will Leave" and "Will Stay" employees.
ml_models.py — SVC.fit() SGD Training
def fit(self, X, y):
    y_pm = np.where(y == 1, 1, -1).astype(float)  # SVM needs labels as +1/-1, not 0/1
    lam  = 1.0 / (self.C * n)                    # regularization (same as LR)

    for epoch in range(200):
        idx = rng.permutation(n)                # shuffle order of employees each epoch
        for i in idx:                            # update weights one employee at a time
            xi, yi, wi = X[i], y_pm[i], sw[i]

            margin = yi * (xi @ self.w_ + self.b_)  # margin = y × (w·x + b)
            # If margin ≥ 1: correctly classified with confidence → just regularize
            # If margin < 1: misclassified or in margin zone → fix it

            if margin < 1:                       # hinge loss: wrong or not confident enough
                self.w_ = self.w_ - lr*(2*lam*self.w_ - wi*yi*xi)  # correct + regularize
                self.b_ = self.b_ + lr*wi*yi    # adjust bias
            else:                                # correct and confident enough
                self.w_ = self.w_ - lr*(2*lam*self.w_)  # only regularize

    # Probability calibration: SVM gives a score, not a probability
    # We fit a Logistic Regression on the SVM scores to convert to probabilities
    scores = X @ self.w_ + self.b_
    self._lr_model = LogisticRegression()
    self._lr_model.fit(scores.reshape(-1, 1), y)  # learn score → probability mapping
Putting It All Together
Cross-Validation — Getting Reliable Scores
Training once and testing once might be lucky or unlucky. Cross-validation trains and tests 5 times on different splits, then averages.

5 scores averaged → reliable estimate. Std deviation shows if model is stable or inconsistent.

ml_models.py — cross_val_score()
def cross_val_score(model, X, y, cv, scoring="recall"):
    scores = []
    for train_idx, test_idx in cv.split(X, y):          # get 5 different train/test splits
        m = model.__class__(**model.get_params())         # create a fresh copy of the model
        m.fit(X[train_idx], y[train_idx])                  # train on this fold's training data
        y_pred = m.predict(X[test_idx])                   # predict on this fold's test data
        scores.append(recall_score(y[test_idx], y_pred))  # record recall for this fold
    return np.array(scores)                               # [0.78, 0.82, 0.79, 0.81, 0.80]
    # In app.py: cv.mean()=0.80 (80% average recall), cv.std()=0.015 (stable)
